# idea/ - Business Context (Phase 1: Constitution)

This directory contains the **business context** discovered from an existing project.

## Purpose

The `idea/` folder stores high-level business information that helps understand:

- What problem the product solves
- Who the target users are
- How the business model works
- Domain-specific terminology

## Expected Files

| File                 | Description                        | Generated By                                      |
| -------------------- | ---------------------------------- | ------------------------------------------------- |
| `business-model.md`  | Business Model Canvas (discovered) | `discovery/phase-1-constitution/business-model-discovery.md` |
| `domain-glossary.md` | Domain terminology and definitions | `discovery/phase-1-constitution/domain-glossary.md`          |

## How to Generate

Run the prompts in `.prompts/discovery/phase-1-constitution/` after completing Phase 0 (Project Onboarding).

### Prerequisites

- `.context/project-config.md` must exist (generated by Phase 0)
- Access to source code repositories
- (Optional) Access to existing documentation (Confluence, Notion, etc.)

## Discovery Sources

Information in this folder is DISCOVERED (not created) from:

1. **Source Code Analysis**
   - Entity/model names reveal domain concepts
   - Module structure reveals business areas
   - Comments and documentation strings

2. **Database Schema**
   - Table names and relationships
   - Field names and data types

3. **Existing Documentation**
   - READMEs, wikis, Confluence pages
   - API documentation
   - User guides

4. **Running Product**
   - UI exploration with Playwright MCP
   - Feature observation

## When to Read This Folder

- **Phase 2+**: Before creating PRD/SRS documentation
- **Phase 5+**: Before writing test cases (understand business context)
- **Any QA work**: To understand domain terminology
